# /draw

**Description**:
Invokes the **Diagram Architect** to visualize the current context or specific text.
See rules: `.cursor/rules/10-diagram-architect.mdc`
See rules: `.cursor/rules/00-router.mdc`
See rules: `.cursor/rules/99-environment.mdc`

**Usage**:
- `/draw`: Visualizes the last message or current file context.
- `/draw <description>`: Creates a diagram based on the description.
- `/draw @file`: Visualizes the logic within a specific file.

**Example**:
`/draw user registration flow with email verification`
